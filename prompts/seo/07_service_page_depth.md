# SEO 07 — Service page depth

**Suggested skill:** `service-page-seo`  
**Suggested agent:** `seo_copy_editor`

## Goal
Expand the core service pages into stronger money pages with FAQs, related links, and clearer conversion structure.

## Scope
Core service pages plus shared service-page template/components.

## Instructions to Codex

1. Read `docs/internal_link_recommendations.csv`.
2. For each core service page, add or improve:
   - intro paragraph
   - what this service solves
   - what is included / process
   - why Shipwrecked
   - FAQ section
   - related service links
   - related location links
   - closing CTA
3. Keep the current design system and component style.
4. Prefer shared components if the stack supports them cleanly.
5. If a human proof section is missing, add a TODO note in `docs/human_inputs_needed.md` rather than inventing proof.


## Done when

- each core service page feels complete
- FAQ and related-links sections exist
- links point to canonical routes


## Constraints

- do not invent prices, guarantees beyond what already exists, or local case studies
- do not add FAQ schema just for search appearance
